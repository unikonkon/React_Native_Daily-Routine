# App Structure — Daily Routine Planner (โครงสร้าง 4 แท็บ)

> เอกสารโครงสร้างแอปฉบับล่าสุด อ้างอิงข้อมูลจริงจาก `Time Table จอย - Time Table.csv`
> ใช้คู่กับ **`SYSTEM_DESIGN_V2.md`** (สถาปัตยกรรม + data model + อัลกอริทึม) — ทั้งคู่แทนที่ `SYSTEM_DESIGN.md` เดิม

## 1. ภาพรวม Tab Bar (4 หน้า)

```
┌─────────────────────────────────────────────────┐
│                   เนื้อหาแต่ละแท็บ                  │
│                                                 │
├─────────────────────────────────────────────────┤
│  📋 กิจกรรม   ➕ เพิ่ม   📊 สรุป   ⚙️ ตั้งค่า      │
└─────────────────────────────────────────────────┘
```

| แท็บ | ชื่อ | หน้าที่ |
|---|---|---|
| 1 | **กิจกรรม** (Home) | แสดงกิจกรรมที่ต้องทำ — มุมมอง วัน / สัปดาห์ / เดือน + แก้ไข / ลบ |
| 2 | **เพิ่มกิจกรรม** | Flow 3 ขั้น: เลือกหมวด → ใส่ชื่อ/รายละเอียด → กำหนดวันเวลา + แจ้งเตือน |
| 3 | **สรุป** | สลับ 2 โหมด: วันว่างที่เหลือ / สรุปเคสที่นัด |
| 4 | **ตั้งค่า** | สถิติ · จัดการหมวดหมู่ · แจ้งเตือน · ธีม · ข้อมูล (Export/Import/Google Sheets) |

## 2. หมวดกิจกรรม (Categories)

จากการวิเคราะห์ CSV ได้ 6 หมวดกิจกรรม + 1 ระบบ priority:

| # | หมวด | สี (แนะนำ) | ตัวอย่างจาก CSV |
|---|---|---|---|
| 1 | 🌅 กิจวัตรประจำวัน (Fixed Routine) | เทา-ฟ้า | WAKE UP + GET DRESS, eat BF / Shake Bodykey, อาบน้ำ-บำรุงผิว, go to bed |
| 2 | 💼 งานประจำ (Work) | น้ำเงิน | work / WORK (09:00–18:00) |
| 3 | 🏋️ ออกกำลังกาย / สุขภาพ (Health) | เขียว | Arobic dance, weight, fitness |
| 4 | 👥 งานธุรกิจ / ทีม (Business & Team) | ส้ม | นัดเคส/เพิ่มเพื่อน, เคส BM, ตอบแชท MLAB, center bkk, QC Center, BEYOND, community, Bridge Online |
| 5 | 📚 พัฒนาตนเอง / เรียนรู้ (Learning) | เหลือง | ฟังลิ้งก์, อ่านหนังสือ, Book club online |
| 6 | 📌 งานอื่น ๆ / นัดพิเศษ (Other) | ม่วง | ไป Survey มศว. |

### ⚠️ ข้อสรุปการวิเคราะห์ — "หมวด 7" ไม่ใช่หมวดกิจกรรม

**ระบบจัดลำดับความสำคัญของเคส** (Legend ท้ายตาราง Excel) ไม่ใช่ประเภทกิจกรรม
แต่เป็น **คุณสมบัติ (attribute) ของเคสในหมวด 4** — จึงออกแบบเป็น **ป้าย Priority 6 ระดับ**
ที่จะโผล่ให้เลือก *เฉพาะตอนเพิ่มกิจกรรมหมวด 4 (งานธุรกิจ/ทีม)* แทนการเป็นหมวดที่ 7:

| ระดับ | ป้าย | สีป้าย |
|---|---|---|
| P1 | คนใหม่ / มีปัญหา — **1st Priority** | แดง |
| P2 | คนเริ่มต้นทำงาน | ส้ม |
| P3 | คนเก่า ที่เวิร์ก | เหลือง |
| P4 | คนเก่า ที่ทำงานด้วย | เขียว |
| P5 | B- | ฟ้า |
| P6 | Member | เทา |

> ผลพลอยได้: หน้า "สรุปเคส" (แท็บ 3) เรียงและกรองตาม priority ได้ทันที

## 3. แท็บ 1 — 📋 กิจกรรม (แสดง / แก้ไข / ลบ)

### 3.1 โครงหน้าจอ

```
┌──────────────────────────────────┐
│  ‹ เมษายน 2026 ›      [วัน|สัปดาห์|เดือน]   ← segmented control
├──────────────────────────────────┤
│  โหมด "วัน" (ค่าเริ่มต้น = วันนี้)      │
│  ┌────────────────────────────┐  │
│  │ 06:00 🌅 WAKE UP+GET DRESS │  │  ← timeline แนวตั้ง
│  │ 08:30 🌅 eat BF/Shake      │  │     บล็อกสีตามหมวด
│  │ 09:00 💼 work              │  │
│  │ ──────── ตอนนี้ 14:02 ───────│  │  ← เส้นเวลาปัจจุบัน
│  │ 18:00 🏋️ Arobic dance      │  │
│  │ 21:00 👥 ตอบแชท MLAB [P2]  │  │  ← เคสมีป้าย priority
│  │ 01:00 🌅 go to bed         │  │
│  └────────────────────────────┘  │
└──────────────────────────────────┘
```

### 3.2 มุมมอง 3 ระดับ

| โหมด | รูปแบบ | การใช้งาน |
|---|---|---|
| **วัน** | Timeline แนวตั้ง 06:00 → 02:30 (ข้ามเที่ยงคืน) | swipe ซ้าย/ขวา เปลี่ยนวัน |
| **สัปดาห์** | ตาราง 7 คอลัมน์ × แถวเวลา (แบบ Excel เดิม) | swipe เปลี่ยนสัปดาห์, แตะช่องดูรายละเอียด |
| **เดือน** | ปฏิทิน + จุดสีบอกความหนาแน่นกิจกรรม | แตะวัน → กระโดดไปโหมด "วัน" ของวันนั้น |

### 3.3 แก้ไข / ลบ

- **แตะกิจกรรม** → Bottom Sheet รายละเอียด: ชื่อ, หมวด, เวลา, การทำซ้ำ, แจ้งเตือน, โน้ต
  - ปุ่ม `[✓ ทำแล้ว]` `[✎ แก้ไข]` `[⏭ เลื่อนนัด]`* `[🗑 ลบ]`
  - *ปุ่ม **เลื่อนนัด** แสดงเฉพาะกิจกรรมหมวด 4 (งานธุรกิจ/ทีม) — เข้าสู่ Reschedule Flow (ดู 3.4)
- **ติ๊ก "ทำแล้ว"** — เปลี่ยน status เป็น `done` ใช้เป็นฐานคำนวณ % สำเร็จ/streak ในหน้าสถิติ
- **แก้ไข** → เปิดฟอร์มเดียวกับ "เพิ่มกิจกรรม" (แท็บ 2) แบบ pre-fill ข้อมูลเดิม
- **ลบ** → ถ้าเป็นกิจกรรมทำซ้ำ ให้เลือก:
  - `ลบเฉพาะครั้งนี้` — ลบ instance เดียว
  - `ลบทั้งชุด` — ลบทุก instance ที่เหลือของกิจกรรมนี้
- **Swipe ซ้ายบนรายการ** → ปุ่มลบด่วน (มี confirm)

### 3.4 Reschedule Flow — เลื่อนนัด (เฉพาะหมวด 4) ⭐

ยกมาจาก `SYSTEM_DESIGN.md` (F3) — เข้าจากปุ่ม `[⏭ เลื่อนนัด]` ใน Bottom Sheet:

```
[นัดเดิม] → ระบุเหตุผล (optional) + เลือกช่วงค้นหา
              ○ ภายใน 3 วัน ○ สัปดาห์นี้ ○ สัปดาห์หน้า ○ กำหนดเอง
          → ระบบสแกนตารางด้วย Free Slot Engine (ตัวเดียวกับแท็บ 3)
          → แสดง list เวลาว่าง เรียงตามความเหมาะสม
              ✦ แนะนำ: เวลาเดียวกับนัดเดิม / วันใกล้สุดก่อน / เคส P1 ดัน slot เร็วสุดขึ้นก่อน
          → เลือก slot → ยืนยัน
          → นัดเดิม mark "rescheduled" + สร้างนัดใหม่ + ย้ายแจ้งเตือน + บันทึก RescheduleLog
```

- ไม่มี slot ว่างในช่วงที่เลือก → เสนอขยายช่วงค้นหา
- จำนวนครั้งที่เลื่อนต่อเคส เก็บใน `RescheduleLog` → โชว์ในประวัติเคส (5.2)

## 4. แท็บ 2 — ➕ เพิ่มกิจกรรม (Flow 3 ขั้น)

```
ขั้น 2.1 เลือกหมวด → ขั้น 2.2 ชื่อ+รายละเอียด → ขั้น 2.3 วันเวลา+แจ้งเตือน → [บันทึก]
   (grid 6 หมวด)      (ฟอร์มปรับตามหมวด)        (ฟอร์มปรับตามหมวด)
```

มี progress indicator บอกขั้นตอน (● ○ ○) — ย้อนกลับแก้ขั้นก่อนหน้าได้เสมอ
ปุ่ม **บันทึก** กดได้เมื่อกรอกครบขั้นต่ำ: หมวด + ชื่อ + วันเวลา

### 4.1 ขั้น 2.1 — เลือกหมวด

UI แบบ **grid การ์ด 2 คอลัมน์ × 3 แถว** (ไอคอน + ชื่อหมวด + สีหมวด)
เลือกแล้วไปขั้น 2.2 อัตโนมัติ

### 4.2 ขั้น 2.2 — ชื่อ + รายละเอียด (ปรับตามหมวด)

ฟิลด์พื้นฐานทุกหมวด: **ชื่อกิจกรรม*** (บังคับ) + **รายละเอียด** (optional)
ฟิลด์เสริมเปลี่ยนตามหมวดที่เลือกในขั้น 2.1:

| หมวด | ฟิลด์เสริมในขั้น 2.2 | ตัวช่วยพิมพ์เร็ว (quick-pick chips) |
|---|---|---|
| 1 กิจวัตรประจำวัน | — | "ตื่นนอน+แต่งตัว", "อาหารเช้า/Shake", "อาบน้ำ-บำรุงผิว", "เข้านอน" |
| 2 งานประจำ | สถานที่ (optional) | "work" |
| 3 ออกกำลังกาย | ประเภท: คาร์ดิโอ / เวท / คลาส | "Arobic dance", "Weight", "Fitness" |
| 4 งานธุรกิจ/ทีม | **คนที่นัด — เลือกจากสมุดรายชื่อ** (multi-select chip, ปุ่ม "+ เพิ่มรายชื่อใหม่") + ช่องทาง: ออนไลน์/เจอตัว — priority ของนัดดึงจากรายชื่อที่เลือก (นัดกลุ่มใช้ค่าสูงสุด) | "นัดเคส", "ตอบแชท MLAB", "center bkk", "QC Center", "เคส BM" |
| 5 พัฒนาตนเอง | สื่อ: หนังสือ / ลิ้งก์เสียง / คลาสออนไลน์ | "อ่านหนังสือ", "ฟังลิ้งก์", "Book club" |
| 6 งานอื่น ๆ | สถานที่ (optional) | — |

> Quick-pick chips ดึงจากชื่อกิจกรรมที่เคยใช้บ่อยในหมวดนั้น (เรียนรู้จากข้อมูลจริง) — แตะแล้วเติมชื่อให้เลย ลดการพิมพ์

**สมุดรายชื่อ (Contacts):** รายชื่อคนที่นัดเก็บเป็นข้อมูลถาวร เลือกซ้ำได้ทุกนัด — แต่ละรายชื่อมี: ชื่อ, ระดับ priority (P1–P6), ช่องทางติดต่อ (เบอร์/LINE, optional) — เพิ่มรายชื่อใหม่ได้จากใน flow นี้เลยไม่ต้องออกไปหน้าอื่น และดูประวัตินัดต่อคนได้จากแท็บ 3 โหมดสรุปเคส (5.2)

### 4.3 ขั้น 2.3 — วันและเวลา (ค่าเริ่มต้นปรับตามหมวด)

**ส่วนที่ 1 — เวลา:** เวลาเริ่ม + เวลาจบ (snap ทีละ 15 นาที)

**ส่วนที่ 2 — รูปแบบการลง (Repeat):**

```
○ ครั้งเดียว          → เลือกวันที่เดียว
○ ทุกวัน
○ เฉพาะเสาร์-อาทิตย์
○ ไม่เอาเสาร์-อาทิตย์ (จันทร์–ศุกร์)
○ เลือกวันเอง         → ติ๊ก จ อ พ พฤ ศ ส อา
```

**ส่วนที่ 3 — ระยะเวลาที่ลง (Horizon):** *(แสดงเมื่อเลือกแบบทำซ้ำ)*

```
[ 1 สัปดาห์ ] [ 2 สัปดาห์ ] [ 1 เดือน ] [ 6 เดือน ] [ 1 ปี ]
เริ่มตั้งแต่วันที่: (ค่าเริ่มต้น = วันนี้, เปลี่ยนได้)
```

**ค่าเริ่มต้นอัจฉริยะตามหมวด** (ผู้ใช้เปลี่ยนได้ทุกค่า):

| หมวด | Repeat เริ่มต้น | Horizon เริ่มต้น | เวลาเริ่มต้น |
|---|---|---|---|
| 1 กิจวัตรประจำวัน | ทุกวัน | 1 เดือน | — (ให้เลือก) |
| 2 งานประจำ | ไม่เอาเสาร์-อาทิตย์ | 1 เดือน | 09:00–18:00 |
| 3 ออกกำลังกาย | เลือกวันเอง | 1 เดือน | 18:00–19:00 |
| 4 งานธุรกิจ/ทีม | **ครั้งเดียว** (นัดเป็นครั้ง ๆ) | — | — (ให้เลือก) |
| 5 พัฒนาตนเอง | ทุกวัน | 2 สัปดาห์ | 22:30–23:00 |
| 6 งานอื่น ๆ | ครั้งเดียว | — | — (ให้เลือก) |

**ส่วนที่ 4 — แจ้งเตือน (Notification):**

```
แจ้งเตือน            [เปิด ●──]   ← ค่าเริ่มต้น: เปิด
เตือนล่วงหน้า        [ 30 นาที ▾]  ← ค่าเริ่มต้น: 30 นาที
   ตัวเลือก: 15 / 30 / 45 / 60 นาที (สเต็ปละ 15 นาที)
   + "กำหนดเอง..." → ใส่ตัวเลขนาทีเองได้
```

- ปิดได้รายกิจกรรม (toggle)
- กิจกรรมทำซ้ำ = ตั้งแจ้งเตือนให้ทุก instance ที่สร้างในช่วง horizon
- ใช้ `expo-notifications` (local notification)

**⚠️ ข้อควรระวังเชิงเทคนิค:** การลงแบบ "ทุกวัน × 1 ปี" = 365 instance — ควรเก็บเป็น **recurrence rule** (กฎการทำซ้ำ) แล้ว generate instance ล่วงหน้าแบบ rolling window (เช่น 60 วัน) แทนการสร้างทั้งหมดทีเดียว เพราะ iOS จำกัด local notification ค้างไว้ได้ ~64 รายการ — ระบบต้อง re-schedule แจ้งเตือนเฉพาะช่วงใกล้ ๆ แล้วเติมใหม่เมื่อเปิดแอป

## 5. แท็บ 3 — 📊 สรุป (2 โหมด สลับด้วยปุ่ม)

```
┌──────────────────────────────────┐
│   [ 🟢 วันว่าง ]  [ 👥 สรุปเคส ]    ← toggle button
│   ‹  ช่วงเวลา  ›   [วัน|สัปดาห์|เดือน]
├──────────────────────────────────┤
│            เนื้อหาตามโหมด           │
└──────────────────────────────────┘
```

### 5.1 โหมด "วันว่าง" — เน้นเวลาว่างที่เหลือ

- คำนวณจากตาราง: ช่วงเวลา 06:00–02:30 ที่ **ไม่มีกิจกรรม** = ว่าง
- **การแสดง:** ช่วงว่าง = บล็อกเขียวสว่างเด่น / กิจกรรมที่มี = บล็อกสีจาง (ยังอ่านได้ แต่ไม่แย่งสายตา)
- มุมมอง:
  - **วัน** — timeline โชว์ slot ว่างเป็นช่วง ๆ พร้อมความยาว เช่น "ว่าง 14:00–18:00 (4 ชม.)"
  - **สัปดาห์** — ตาราง 7 วัน ไฮไลต์ช่องว่าง + สรุปหัวคอลัมน์ "ว่างรวม x ชม."
  - **เดือน** — ปฏิทิน ไล่เฉดสีเขียวตามจำนวนชั่วโมงว่างของแต่ละวัน (ยิ่งว่างยิ่งเขียว)
- ใช้ **Free Slot Engine** ตัวเดียวกับที่ออกแบบไว้ใน `SYSTEM_DESIGN.md` (F3)
- แตะ slot ว่าง → shortcut "➕ ลงกิจกรรมช่องนี้" (ส่งวัน+เวลาไป pre-fill แท็บ 2)

### 5.2 โหมด "สรุปเคส" — เฉพาะหมวด 4 (งานธุรกิจ/ทีม)

- กรองเฉพาะกิจกรรมหมวด **งานธุรกิจ/ทีม** ที่มีลักษณะเป็นนัด (มีชื่อเคส)
- แสดงเป็น **list จัดกลุ่มตามช่วงเวลา** (วัน / สัปดาห์ / เดือน):

```
  วันพุธ 8 เม.ย.
  ├ 18:00  นัดเคส คุณเอ         [P1 คนใหม่/มีปัญหา]  ⏳ รอถึงนัด
  ├ 19:00  center bkk                              ✅ ทำแล้ว
  สัปดาห์นี้รวม: นัด 5 · ทำแล้ว 2 · เลื่อน 1
```

- แถบสรุปตัวเลขบนหัว: จำนวนนัดทั้งหมด / ทำแล้ว / ยังไม่ถึง / เลื่อน-ยกเลิก
- กรองตาม **priority (P1–P6)** ได้ — ค่าเริ่มต้นเรียง P1 ขึ้นก่อน
- แตะรายการ → Bottom Sheet เดียวกับแท็บ 1 (แก้ไข/ลบ/เลื่อนนัดได้จากที่นี่เลย)
- **แตะชื่อคน → หน้าประวัติเคสรายคน:** นัดครั้งล่าสุดเมื่อไหร่, นัดทั้งหมดกี่ครั้ง, เลื่อนไปกี่ครั้ง (จาก RescheduleLog), ปุ่ม "นัดใหม่" (pre-fill คนนี้ไปแท็บ 2) + แก้ไขรายชื่อ/priority ได้

## 6. แท็บ 4 — ⚙️ ตั้งค่า (5 เมนู)

```
⚙️ ตั้งค่า
├── 1. 📈 สถิติกิจกรรม
├── 2. 🗂 จัดการหมวดหมู่
├── 3. 🔔 ตั้งค่าแจ้งเตือน
├── 4. 🎨 ธีม
└── 5. 💾 ข้อมูล (Export / Import / Google Sheets)
```

### 6.1 สถิติกิจกรรม

- ช่วงเวลา: `[สัปดาห์นี้] [เดือนนี้] [ปีนี้] [เลือกช่วงวันที่เอง…]` (date range picker)
- สิ่งที่แสดง:
  - % ทำสำเร็จ (done / planned) + streak ทำต่อเนื่อง
  - กราฟแท่ง: จำนวนชั่วโมงต่อหมวดหมู่ (สีตามหมวด)
  - กราฟเส้น: แนวโน้มจำนวนกิจกรรมต่อวัน
  - สถิติเคส: จำนวนนัดต่อช่วง + แยกตาม priority

### 6.2 จัดการหมวดหมู่

- รายการ 6 หมวดเริ่มต้น — **เพิ่ม / แก้ไข / ลบ** ได้อิสระ
- แก้ไขได้: ชื่อ, สี (color picker), ไอคอน (emoji picker), ค่าเริ่มต้นขั้น 2.3 (repeat/horizon/เวลา)
- ลบหมวดที่มีกิจกรรมอยู่ → ถามให้ย้ายกิจกรรมไปหมวดอื่นก่อน
- ส่วนย่อย: **จัดการระดับ Priority ของเคส** (แก้ชื่อป้าย/สี P1–P6 ได้)

### 6.3 ตั้งค่าแจ้งเตือน

- สวิตช์รวมทั้งแอป (master toggle)
- ค่าเริ่มต้นเตือนล่วงหน้า (default 30 นาที) — ใช้กับกิจกรรมใหม่
- สรุปเช้าประจำวัน (optional): แจ้ง 06:00 "วันนี้มี x กิจกรรม นัด y รายการ"
- เสียง/การสั่น

### 6.4 ธีม

- สว่าง / มืด / ตามระบบ (ค่าเริ่มต้น: ตามระบบ)

### 6.5 ข้อมูล

| ฟังก์ชัน | รายละเอียด |
|---|---|
| **Export** | ไฟล์ CSV (ฟอร์แมตเดียวกับ Excel เดิม — สัปดาห์×เวลา) + JSON (backup เต็มรูปแบบ) → share sheet ของเครื่อง |
| **Import** | อ่านไฟล์ CSV/JSON ที่ export ไว้ → merge หรือแทนที่ (ให้เลือก) |
| **ส่งเข้า Google Sheets** | **ส่งออกทางเดียว** (แอป → Sheets เท่านั้น ไม่ดึงกลับ) — เชื่อมบัญชี Google (OAuth ผ่าน `expo-auth-session`) → สร้าง/อัปเดต spreadsheet ใน Drive ของผู้ใช้ ฟอร์แมตเดียวกับ Time Table เดิม |

> **แผนเป็นเฟส:** เฟสแรกทำ Export/Import ไฟล์ก่อน (ไม่ต้องมี backend) — Google Sheets sync เป็นเฟสถัดไปเพราะต้องตั้งค่า Google Cloud OAuth client และมีเงื่อนไขกับ Expo Go (redirect URI) — รายละเอียดจะตามใน tech spec แยก

## 7. Data Model (ปรับจาก SYSTEM_DESIGN.md ให้ตรง flow ใหม่)

```
┌────────────┐      ┌───────────────────┐      ┌──────────────────┐
│ Category    │1    *│ Activity           │1    *│ ActivityInstance  │
│─────────────│◄─────│───────────────────│◄─────│──────────────────│
│ id          │      │ id                 │      │ id               │
│ name        │      │ title              │      │ activityId       │
│ color       │      │ detail?            │      │ date             │
│ icon        │      │ categoryId         │      │ startTime        │
│ defaults{}  │──┐   │ startTime/endTime  │      │ endTime          │
└────────────┘  │   │ repeat: none|daily │      │ status: planned| │
                │   │   |weekend|weekday │      │  done|skipped|   │
 defaults =     │   │   |custom(days[])  │      │  rescheduled|    │
 ค่าเริ่มต้นขั้น 2.3 │   │ horizon: 1w|2w|1m │      │  cancelled       │
 ของหมวดนั้น     │   │   |6m|1y           │      └──────────────────┘
                │   │ startDate          │      ┌──────────────────┐
                └──▶│ notifyEnabled=true │      │ CaseInfo (หมวด 4) │
                    │ notifyBefore=30min │      │──────────────────│
                    │ caseInfo? ────────────────▶ contactIds[] ────────┐
                    └───────────────────┘      │ channel: online| │   │
                                               │  inperson        │   │
                                               └──────────────────┘   │
                                                                      ▼ *
┌─────────────────┐                            ┌──────────────────────┐
│ RescheduleLog    │                            │ Contact (สมุดรายชื่อ)   │
│─────────────────│                            │──────────────────────│
│ id              │                            │ id                   │
│ instanceId      │                            │ name                 │
│ fromDate/Time   │                            │ priority: P1–P6      │
│ toDate/Time     │                            │ phone/line?          │
│ reason?         │                            │ note?                │
│ createdAt       │                            └──────────────────────┘
└─────────────────┘                             * นัดกลุ่มได้หลายคน —
                                                  priority ของนัดใช้ค่าสูงสุดในกลุ่ม
```

หลักการสำคัญ:

- **เก็บ recurrence rule ไม่ใช่ instance ทั้งหมด** — generate `ActivityInstance` แบบ rolling window (60 วันล่วงหน้า) กัน horizon 1 ปีสร้างข้อมูลบวม
- **แก้ไข instance เดียว vs ทั้งชุด** — instance ที่ถูกแก้เดี่ยวจะ mark `detached` ไม่โดน rule ทับ
- **Contact เก็บถาวร** — priority ติดที่ตัวคน (ไม่ใช่ที่นัด) นัดใหม่ของคนเดิมได้ priority อัตโนมัติ, ประวัตินัด/จำนวนครั้งที่เลื่อน query จาก ActivityInstance + RescheduleLog
- **วันของแอปเริ่ม 04:00 → 03:59** วันถัดไป (รองรับ go to bed 01:00 นับเป็นคืนของวันเดิม ตาม Excel)
- เวลาทุกจุด snap **15 นาที**
- Storage: AsyncStorage (เฟสแรก) → SQLite เมื่อข้อมูล/สถิติโต — ตาม `PROJECT_PLAN.md`
